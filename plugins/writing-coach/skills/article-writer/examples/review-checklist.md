# Article Review Checklist

Use this checklist when reviewing a draft article.

---

## Pre-Review Setup

Before reviewing, establish:

- [ ] What is the stated goal of this article?
- [ ] What pattern is being used? (Bug Hunt, Lessons Learned, etc.)
- [ ] Who is the target reader?

---

## The 3 F's: Core Review

### Facts Check (Skill 8)

For each claim in the article:

- [ ] Is every arguable statement supported by evidence?
- [ ] Are there specific numbers where relevant?
- [ ] Are there code examples where helpful?
- [ ] Would a skeptical reader trust this?
- [ ] Are sources cited where needed?

**Claims to flag:**
```
Page/Section: ________________
Claim: ________________
Missing: ________________
```

### Focus Check (Skill 9)

For each section/paragraph:

- [ ] Does every paragraph advance the stated goal?
- [ ] Is there any off-topic content that can be cut?
- [ ] Is there unnecessary background or context?
- [ ] Is there showing-off that doesn't serve the reader?

**Content to cut:**
```
Page/Section: ________________
Why cut: ________________
```

### Flow Check (Skill 10)

For overall structure:

- [ ] Is there logical progression from start to end?
- [ ] Are transitions smooth between sections?
- [ ] Would a reader get lost anywhere?
- [ ] Does each section connect to what came before?

**Flow issues:**
```

Issue: ________________
Suggested fix: ________________
```

## Pattern-Specific Check (Skill 12)

Based on the article's pattern, check:

**Bug Hunt:**
- [ ] Opens with symptoms, not solution?
- [ ] Includes false leads?
- [ ] Has clear breakthrough moment?
- [ ] Explains why the fix works?

**Rewrote It in X:**
- [ ] Clear before/after comparisons?
- [ ] Concrete metrics?
- [ ] Honest about challenges?
- [ ] Discusses who should/shouldn't do this?

**How We Built It:**
- [ ] Architecture clearly explained?
- [ ] Explains WHY, not just WHAT?
- [ ] Tradeoffs discussed?
- [ ] Limitations acknowledged?

**Lessons Learned:**
- [ ] Each lesson grounded in experience?
- [ ] Actionable recommendations?
- [ ] Honest about mistakes?
- [ ] Not preachy?

**Thoughts on Trends:**
- [ ] Author's credibility established?
- [ ] Clear position taken?
- [ ] Counterarguments addressed?
- [ ] Evidence supports claims?

**Non-markety Product:**
- [ ] Valuable without the product?
- [ ] No marketing language?
- [ ] Product mention is natural?
- [ ] Company affiliation disclosed?

**Benchmarks:**
- [ ] Methodology explained?
- [ ] Results reproducible?
- [ ] No exaggeration in charts?
- [ ] Limitations acknowledged?

---

## Writing Quality

### Conversational Voice (Skill 4)
- [ ] Sounds like a human, not a robot?
- [ ] Uses "I/we" and "you" appropriately?
- [ ] Authentic - shares struggles and uncertainties?

### Paragraph Clarity (Skill 5)
- [ ] One idea per paragraph?
- [ ] No overly long sentences?
- [ ] Action verbs used?

### Scannability (Skill 6)
- [ ] Headings every 2+ scrolls?
- [ ] No massive text blocks?
- [ ] Lists and tables used where helpful?
- [ ] Key points bolded?

### Show Don't Tell (Skill 7)
- [ ] Code examples where helpful?
- [ ] Diagrams where helpful?
- [ ] Specific numbers, not vague claims?

---

## Pre-Publish Verification (Skill 13)

- [ ] Title renders well in preview?
- [ ] All links work?
- [ ] Code is copy-pasteable?
- [ ] Images properly sized?
- [ ] Meta description has keywords?
- [ ] URL is clean?

---

## Self-Review Test (Skill 14)

- [ ] Would I share this if someone else wrote it?
- [ ] Can a skeptical reader trust the claims?
- [ ] Does every section advance the stated goal?
- [ ] Is the key takeaway clear early?
- [ ] Have I shown work, not just conclusions?
